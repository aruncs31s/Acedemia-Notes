#### *1. Frequency Reuse & Capacity*

- **Reuse patterns**: N = 1, 3, 4, 7, 9, 12, 13, 19
- **Cluster size (N)**: Number of cells per cluster
- **Capacity scaling**: Capacity ∝ N (for fixed spectrum)

**Key Formulas:**
- Reuse Distance: $D = R\sqrt{3N}$
- SIR: $SIR = \frac{1}{6(N/K_{reuse}-1)}$ (worst case)

>This concept revolutionized cellular communications by enabling scalable capacity growth as demand increased, forming the foundation of modern mobile networks.

#### *2. Cell Splitting & Sectoring*

- **Cell Splitting**: Adding smaller cells to increase capacity
  - Triggers: Traffic demand, QoS issues
  - Increases: Capacity ∝ 1/R²
  - Challenge: More handoffs, increased interference

- **Sectoring**: Dividing cells into 3-6 sectors
  - Reduces: Co-channel interference
  - Increases: Reuse efficiency
  - Types: 120° (3-sector), 60° (6-sector)

#### *3. Trunking & Grade of Service*

- **Trunking Principle**: Pooled resources serve more users
- **Erlang B Formula**: $B(C,A)=\dfrac{A^C/C!}{\sum_{k=0}^C A^k/k!}$
- **GOS (Grade of Service)**: Blocking probability target (typically 1-2%)
- **Traffic Intensity**: Erlangs = calls/hour × holding time / hour

#### *5. Fading (Small-Scale / Multipath)*

- **Large-Scale Fading**: Path Loss + Shadowing (log-normal)
- **Small-Scale Fading**: Rapid fluctuations due to multipath

**Types:**
| Based on | Flat | Frequency-Selective |
|----------|------|-------------------|
| **Delay Spread** | $B_s < B_c$ | $B_s > B_c$ |
| Based on | Slow | Fast |
| **Doppler** | $T_s < T_c$ | $T_s > T_c$ |

**Statistical Models:**
- **Rayleigh** (NLOS): $f_R(r) = \frac{r}{\sigma^2}e^{-r^2/2\sigma^2}$
- **Rician** (LOS): Parameter K = power(LOS)/power(scattered)

#### *6. Doppler / Time Selectivity*

- **Doppler Shift**: $f_d = \frac{v}{\lambda}\cos\theta = f_m\cos\theta$
- **Max Doppler**: $f_m = \frac{v}{\lambda} = \frac{v f_c}{c}$
- **Coherence Time**: $T_c \approx \frac{0.423}{f_m}$ (channel constant time)

**Practical Examples:**
| Speed | $f_c$ | $f_m$ | $T_c$ |
|-------|-------|-------|-------|
| 3 km/h | 2 GHz | 5.5 Hz | 77 ms |
| 60 km/h | 2 GHz | 111 Hz | 3.8 ms |
| 300 km/h | 2 GHz | 556 Hz | 0.76 ms |

#### *7. Outage Probability*

- **Definition**: Probability SNR falls below threshold ($\rho_{min}$)
- **Rayleigh**: $P_{outage} = 1 - e^{-\rho_{min}/\bar{\rho}}$
- **Fade Margin**: Extra power to achieve target outage (typically 10-20 dB)

**Capacity with Outage:**
$$C_{outage} = (1-P_{outage}) \times B \times \log_2(1+\rho_{min})$$

#### *8. Link Budget / Friis Equation*

**Friis Formula:**
$$P_r = P_t G_t G_r \left(\frac{\lambda}{4\pi d}\right)^2$$

**Link Budget Steps:**
1. Transmit Power ($P_t$)
2. + Antenna Gains ($G_t, G_r$)
3. - Path Loss ($PL_{dB}$)
4. - Misc. Losses (cables, feed, etc.)
5. = Received Power ($P_r$)

**Path Loss:**
- Free Space: $PL_{dB} = 20\log_{10}(4\pi d/\lambda)$
- Two-Ray (far): exponent = 4

#### *10. Equalization (Linear vs Nonlinear / ZF)*

**Linear Equalizers:**
| Type | Pros | Cons |
|------|------|------|
| **ZF** | Eliminates ISI completely | Amplifies noise |
| **MMSE** | Minimizes MSE | Residual noise |

**Non-Linear Equalizers:**
- **DFE** (Decision Feedback): Better for severe ISI
- **MLSE** (Maximum Likelihood): Optimal but complex

**When to Use:**
- Mild ISI: Linear (ZF/MMSE)
- Severe ISI: DFE/MLSE
- High SNR: ZF preferred

#### *11. Cyclic Prefix / OFDM Fundamentals*

**OFDM Key Concepts:**
- **Orthogonality**: $\Delta f = \frac{1}{T_u}$ (subcarrier spacing)
- **CP Length**: $T_{CP} \geq \tau_{max}$ (max delay spread)
- **Why CP?** Converts linear convolution → circular → enables simple FFT equalization

**Advantages:**
- Resists ISI (if CP > max delay spread)
- Efficient FFT implementation
- Handles frequency-selective fading

**Disadvantages:**
- PAPR problem
- Sensitive to frequency offset
- CP overhead (~20%)

#### *12. PAPR in OFDM*

**Definition:**
$$\text{PAPR} = \frac{\max_t |x(t)|^2}{\mathbb{E}[|x(t)|^2]}$$

**Problem:** High PAPR → PA needs large linear range → inefficiency

**Reduction Techniques:**
| Technique | Pros | Cons |
|-----------|------|------|
| **Clipping** | Simple | Spectral growth |
| **Coding** | Good coding gain | Data rate loss |
| **PTS** | Flexible | Side info needed |
| **ACE** | Good performance | Complex |

#### *13. Channel Capacity (Shannon / Ergodic)*

**Shannon Capacity (AWGN):**
$$C = B \log_2(1 + SNR)$$

**Ergodic Capacity (Fading):**
$$C = \mathbb{E}_h[B \log_2(1 + SNR \cdot |h|^2)]$$

**With Outage:**
- Outage probability: $P_{out} = P(SNR < \rho_{min})$
- Capacity: $C_{outage} = (1-P_{outage})B\log_2(1+\rho_{min})$

**Key Insights:**
- High SNR: Capacity ∝ log₂(SNR) (logarithmic growth)
- Low SNR: $C \approx \frac{B}{\ln 2}SNR$ (nearly linear!)
- Bandwidth-limited: Double B → Double C

#### *14. Multiple Access Comparison*

| Method | Resource Axis | Key Limitation | Features |
|--------|--------------|---------------|----------|
| **FDMA** | Frequency | Guard bands needed | Simple, continuous |
| **TDMA** | Time | Sync required | Efficient, bursty |
| **CDMA** | Code | MAI (self-interference) | Soft capacity |
| **OFDMA** | Frequency (subcarriers) | PAPR | Multi-user OFDM |

**Key Trade-offs:**
- FDMA: Guard bands → spectral waste
- TDMA: Sync overhead, idle time
- CDMA: Near-far problem, complex power control
- OFDMA: Best for broadband, but PAPR issue

#### *16. Ionospheric Propagation*

**Critical Frequency:**
$$f_c = 9\sqrt{N_{max}}$$ (MHz)

**Key Concepts:**
- **Layers**: D, E, F1, F2 (F2 most important)
- **Skip Distance**: Min distance for refraction
- **MUF** (Max Usable Frequency): > f_c for escape

**Refraction Mechanism:**
- Refractive index: $n = \sqrt{1 - \frac{80.6N}{f^2}}$
- Wave bends when entering ionosphere
- Bends more at lower frequencies

#### *17. Two-Ray / Path Loss Models*

**Free Space (d < d_bp):**
- Power decay: $P_r \propto 1/d^2$
- Path loss exponent: 2
- Condition: Clear LOS

**Two-Ray (d > d_bp):**
- Power decay: $P_r \propto 1/d^4$
- Path loss exponent: 4
- Condition: Ground reflection dominant

**Breakpoint Distance:**
$$d_{bp} = \frac{4 h_t h_r}{\lambda}$$

#### *18. Trunking / Erlang Formulas*

**Erlang B Formula:**
$$B(C,A) = \frac{A^C/C!}{\sum_{k=0}^C \frac{A^k}{k!}}$$

**Quick Estimation:**
| GOS Target | Erlangs per channel |
|------------|-------------------|
| 1% | ~0.8 |
| 2% | ~1.0 |
| 5% | ~1.3 |

**Efficiency:**
$$\eta = \frac{A}{C} \times (1 - B) \times 100\%$$
