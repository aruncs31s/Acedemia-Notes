# 4.2 Equalization

Equalization is a technique used to combat intersymbol interference (ISI) caused by multipath fading in wireless channels. The equalizer is essentially an inverse filter of the channel.

## Need for Equalization

In wireless channels, transmitted signals arrive at the receiver through multiple paths with different delays. This causes:

- ** ISI**: Symbols overlap with each other, causing distortion
- ** Frequency Selectivity**: Different frequency components experience different attenuation
- ** Time Dispersion**: Channel response spreads in time

Equalizers reverse this effect by filtering the received signal.

## Linear Equalizers

Linear equalizers use FIR (Finite Impulse Response) or IIR (Infinite Impulse Response) filters to equalize the channel.

### Zero Forcing (ZF) Equalizer

The Zero Forcing equalizer inverts the channel response exactly to eliminate ISI.

- **Principle**: Equalizer response = 1 / H(f), where H(f) is the channel frequency response
- **Advantages**:
  - Removes ISI completely (in theory)
  - Simple to implement
- **Disadvantages**:
  - Amplifies noise, especially when channel has deep fades (1/H(f) becomes large)
  - Unstable if channel has zeros near unit circle
- **Output**: y(t) = x(t) + n(t) [where n(t) is amplified noise]
- **Best for**: Channels with good SNR, where noise amplification is acceptable

### Minimum Mean Square Error (MMSE) Equalizer

The MMSE equalizer minimizes the mean square error between transmitted and received symbols.

- **Principle**: Minimize E[|x(t) - ŷ(t)|²]
- **Equalizer coefficients**: w = (H*H + σ_n²/I)^{-1} H*
- **Advantages**:
  - Balances noise amplification and ISI removal
  - Better performance than ZF in noisy channels
- **Disadvantages**:
  - More complex than ZF
  - May not completely eliminate ISI
- **Applications**: Most practical wireless systems use MMSE or its variants

### Comparison: ZF vs MMSE

| Feature | Zero Forcing | MMSE |
|---------|-------------|------|
| Noise Amplification | High | Low |
| ISI Removal | Complete | Partial |
| Complexity | Low | Medium |
| Performance | Poor at low SNR | Good at all SNR |
| Best For | High SNR channels | Practical systems |

## Non-Linear Equalizers

Non-linear equalizers provide better performance than linear equalizers for severe ISI channels.

### Decision Feedback Equalizer (DFE)

The DFE uses past detected symbols to cancel ISI from current symbols.

- **Structure**:
  - **Forward Filter**: Feeds forward filter to equalize future symbols
  - **Feedback Filter**: Uses detected symbols to cancel ISI from past symbols
- **Advantages**:
  - Better performance than linear equalizers
  - Handles severe ISI cases
- **Disadvantages**:
  - Error propagation: errors can propagate if symbol decision is wrong
  - More complex than linear equalizers
- **Applications**: Used in severe ISI channels like underwater acoustics

### Maximum Likelihood Sequence Estimation (MLSE)

The MLSE finds the most likely transmitted sequence given the received signal.

- **Principle**: Evaluate all possible transmitted sequences, select the one with maximum probability
- **Implementation**: Viterbi algorithm
- **Complexity**: O(2^M) where M is the number of channel taps
- **Advantages**:
  - Optimal performance
  - Best for severe ISI
- **Disadvantages**:
  - Exponential complexity with channel length
  - Impractical for long channels
- **Applications**: Used in GSM with 5-tap channel

## Adaptive Equalization

Adaptive equalizers automatically adjust their coefficients based on the received signal, making them suitable for time-varying channels.

### LMS (Least Mean Squares) Algorithm

The LMS algorithm is the most widely used adaptive algorithm.

#### Algorithm

```
Initialize: w(0) = 0
For each iteration n:
  1. Compute output: y(n) = w^H(n) * x(n)
  2. Compute error: e(n) = d(n) - y(n)
  3. Update coefficients: w(n+1) = w(n) + μ * x(n) * e*(n)
```
Where:
- w(n): Equalizer coefficient vector at time n
- x(n): Received signal vector
- d(n): Desired signal (training sequence)
- μ: Step size (learning rate)

#### Properties

- **Step Size (μ)**: 
  - Small μ: Slow convergence, low steady-state error
  - Large μ: Fast convergence, large steady-state error
- **Stability Condition**: 0 < μ < 2/λ_max (where λ_max is largest eigenvalue of autocorrelation matrix)
- **Convergence**: Requires 10-20 iterations per coefficient for convergence
- **Misadjustment**: Steady-state error due to gradient noise

### RLS (Recursive Least Squares) Algorithm

The RLS algorithm provides faster convergence than LMS.

```
Initialize: w(0) = 0, P(0) = δ^-1 * I
For each iteration n:
  1. Compute gain: k(n) = P(n-1) * x(n) / (λ + x^H(n) P(n-1) x(n))
  2. Compute error: e(n) = d(n) - w^H(n-1) x(n)
  3. Update coefficients: w(n) = w(n-1) + k(n) e*(n)
  4. Update inverse correlation: P(n) = (P(n-1) - k(n) x^H(n) P(n-1)) / λ
```

#### LMS vs RLS

| Feature | LMS | RLS |
|---------|-----|-----|
| Convergence Speed | Slow | Fast |
| Complexity | O(N) | O(N²) |
| Steady-state Error | Higher | Lower |
| Stability | Very Stable | May diverge |
| Applications | General use | Fast-tracking needs |

## Training-Based vs Blind Equalization

### Training-Based Equalization

- Uses known training sequences inserted in the transmitted data
- Advantages: Reliable convergence, fast adaptation
- Disadvantages: Reduces data rate (overhead)
- Applications: Most wireless systems (LTE, Wi-Fi, GSM)

### Blind Equalization

- Does not require training sequences
- Examples: Constant Modulus Algorithm (CMA), decision-directed
- Advantages: Higher spectral efficiency
- Disadvantages: Slower convergence, may converge to local minima
- Applications: Some broadcast applications
