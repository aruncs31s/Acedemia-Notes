## 1. Methods of Filtering Long Sequences

> [!question] Question
> What are the methods of filtering a long sequence? Explain.

- The two standard methods to filter a long data sequence block-by-block are **Overlap-Add** and **Overlap-Save**. Both use FFT/IFFT computation to perform block convolution efficiently.
- Detailed explanations of these algorithms and their execution steps can be found in [[Filtering Long Sequences]].

### Brief Summary of Methods
- **Overlap-Add Method:** The input sequence is split into *non-overlapping* blocks. Each block is zero-padded, convolved with the filter, and the overlapping outputs (tails) are added together to assemble the final output.
- **Overlap-Save Method:** The input sequence is split into *overlapping* blocks. Circular convolution is performed on each block using FFT. The first $M-1$ samples (where $M$ is the filter length) of each output block are corrupted by wrap-around and are discarded, while the rest are saved and concatenated.

## 2. Three Properties of DFT

> [!question] Question
> Give any three properties of DFT.

Detailed mathematical proofs and explanations are available in [[DSP Module 1#Properties of DFT and Examples]]. The three commonly used properties are:

1. **Linearity:**
   $$x_1[n] \xleftrightarrow{\text{DFT}} X_1[k] \quad \text{and} \quad x_2[n] \xleftrightarrow{\text{DFT}} X_2[k]$$
   $$a x_1[n] + b x_2[n] \xleftrightarrow{\text{DFT}} a X_1[k] + b X_2[k]$$

2. **Periodicity:**
   Both the sequence $x[n]$ and its DFT $X[k]$ are periodic with period $N$:
   $$x[n] = x[n+N] \quad \text{and} \quad X[k] = X[k+N]$$

3. **Circular Convolution Theorem:**
   Pointwise multiplication of DFTs in the frequency domain corresponds to circular convolution in the time domain:
   $$x[n] \circledast h[n] \xleftrightarrow{\text{DFT}} X[k] \cdot H[k]$$

## 3. 4-Point DFT of $\cos(2\pi n)$ and $\cos(\pi n)$

> [!question] Question
> Calculate the 4-point DFT of $\cos(2\pi n)$ or $\cos(\pi n)$.
> *(Note: Both versions frequently appear in exam papers. Both are solved below.)*

### Case A: For $x[n] = \cos(2\pi n)$
Since $n$ takes integer values ($0, 1, 2, 3$):
$$x[n] = \cos(2\pi n) = 1 \quad \text{for } n = 0, 1, 2, 3$$
The sequence is $x[n] = \{1, 1, 1, 1\}$.

Calculating the 4-point DFT:
$$X[k] = \sum_{n=0}^{3} x[n] W_4^{kn} = \sum_{n=0}^{3} 1 \cdot e^{-j\frac{2\pi}{4}kn}$$
- For $k = 0$: $X[0] = 1 + 1 + 1 + 1 = 4$
- For $k \ne 0$: The sum of orthogonal complex exponentials over a full period is zero.
  - $X[1] = 1 + (-j) + (-1) + (j) = 0$
  - $X[2] = 1 + (-1) + 1 + (-1) = 0$
  - $X[3] = 1 + (j) + (-1) + (-j) = 0$

> [!important] Result for $\cos(2\pi n)$
> $$X[k] = \{4, 0, 0, 0\}$$

### Case B: For $x[n] = \cos(\pi n)$
Since $n$ takes integer values ($0, 1, 2, 3$):
$$x[n] = \cos(\pi n) = (-1)^n \quad \text{for } n = 0, 1, 2, 3$$
The sequence is $x[n] = \{1, -1, 1, -1\}$.

Calculating the 4-point DFT:
$$X[k] = \sum_{n=0}^{3} (-1)^n W_4^{kn}$$
- For $k = 0$: $X[0] = 1 - 1 + 1 - 1 = 0$
- For $k = 1$: $X[1] = 1 - (-j) + 1(-1) - (-j) = 1 + j - 1 - j = 0$
- For $k = 2$: $X[2] = 1 - (-1) + 1(1) - (-1) = 1 + 1 + 1 + 1 = 4$
- For $k = 3$: $X[3] = 1 - (j) + 1(-1) - (j) = 1 - j - 1 + j = 0$

> [!important] Result for $\cos(\pi n)$
> $$X[k] = \{0, 0, 4, 0\}$$

## 4. Circular Time Reversal

> [!question] Question
> Find the circular time reversal of $x[n] = [8, 5, 3, 1]$.

The circular time-reversal of an $N$-point sequence is defined as:
$$y[n] = x[(-n) \bmod N]$$

For $N = 4$ and $x[n] = [8, 5, 3, 1]$:
- $y[0] = x[(-0) \bmod 4] = x[0] = 8$
- $y[1] = x[(-1) \bmod 4] = x[3] = 1$
- $y[2] = x[(-2) \bmod 4] = x[2] = 3$
- $y[3] = x[(-3) \bmod 4] = x[1] = 5$

> [!important] Result
> The circular time-reversed sequence is:
> $$\mathbf{y[n] = [8, 1, 3, 5]}$$

## 5. Butterworth IIR Design Steps

> [!question] Question
> Explain the design steps of an IIR filter using the Butterworth approximation.

A detailed description of the design equations can be found in [[Filters#Bilinear Transform]] and [[Filters#Steps to design a filter using Bilinear Transformation]]. The summary steps are:

1. **Specify Digital Filter Specs:** Identify the ripple requirements ($A_p, A_s$) and cutoff frequencies ($\omega_p, \omega_s$).
2. **Frequency Pre-warping:** Convert the digital edge frequencies to analog using:
   $$\Omega = \frac{2}{T} \tan\left(\frac{\omega}{2}\right)$$
3. **Determine Filter Order ($N$) and Cutoff Frequency ($\Omega_c$):** Calculate using the specifications.
4. **Locate Analog Poles:** Place the poles on the LHP of a circle in the $s$-plane.
5. **Form the Analog Transfer Function $H_a(s)$:** Construct the transfer function from the poles.
6. **Apply Bilinear Transformation:** Map $H_a(s)$ to the digital transfer function $H(z)$ via:
   $$s = \frac{2}{T}\left(\frac{1 - z^{-1}}{1 + z^{-1}}\right)$$

## 6. Advantages of Frequency Sampling Method

> [!question] Question
> What are the advantages of the frequency sampling method in FIR filter design?

The advantages of the frequency sampling method (referenced in [[FIR Filter#What is the advantages of frequency sampling technique in FIR filter design?]]) are:

1. **Arbitrary Magnitude Response:** Allows direct specification of the desired frequency response at uniform frequency samples.
2. **Exact Zeros/Notches:** Enables easy placement of exact zeros or notches at specific frequencies.
3. **Computational Efficiency:** Leverages fast FFT/IFFT operations to find filter coefficients.
4. **Discontinuous Responses:** Highly suitable for filters with irregular or discontinuous frequency profiles.
5. **Guaranteed Linear Phase:** Inherently yields a linear phase response if symmetry constraints are followed.

## 7. Cascade Implementation of IIR

> [!question] Question
> Explain the cascade implementation of IIR systems.

For detailed concepts, see [[Module 4#3. Cascade Realization]].

- **Definition:** The cascade implementation realizes the transfer function $H(z)$ as a product of first-order or second-order sections (biquads) connected in series:
  $$H(z) = H_1(z) \cdot H_2(z) \cdot \dots \cdot H_K(z)$$
- **Signal Flow:** The input goes to the first section, the output of the first goes to the input of the second, and so on:
  $$x[n] \to [H_1(z)] \to [H_2(z)] \to \dots \to y[n]$$
- **Aesthetic/Practical Advantage:** It drastically reduces coefficient quantization sensitivity and round-off noise compared to direct-form implementations, making it numerically robust.

## 8. Linear-Phase FIR Conditions

> [!question] Question
> What is a linear phase FIR filter? What are the conditions to achieve it?

For full derivations, see [[Filters#🧮 Conditions for FIR Filters to Have Linear Phase]].

- **Definition:** A filter is linear-phase if its phase response is a linear function of frequency, resulting in a constant group delay:
  $$\tau_g = -\frac{d\phi(\omega)}{d\omega} = \text{constant}$$
  This preserves the shape of the waveform without phase distortion.
- **Conditions:** For a causal FIR filter of length $N$, linear phase is achieved if the impulse response $h[n]$ is either:
  1. **Symmetric (Even Symmetry):** $h[n] = h[N-1-n]$
  2. **Antisymmetric (Odd Symmetry):** $h[n] = -h[N-1-n]$

## 9. DSP Processor vs General-Purpose Processor

> [!question] Question
> Compare a DSP processor with a General-Purpose Processor (GPP).

| Feature | DSP Processor | General-Purpose Processor (GPP) |
| :--- | :--- | :--- |
| **Architecture** | Typically **Harvard architecture** (separate program and data buses for parallel access). | Typically **Von Neumann architecture** (shared bus for program and data). |
| **MAC Operations** | Contains a dedicated hardware **MAC (Multiply-Accumulate)** unit to perform $a \cdot b + c$ in 1 cycle. | Performs multiplication and addition sequentially using general ALUs (takes multiple cycles). |
| **Addressing Modes** | Supports hardware circular addressing and bit-reversed addressing (essential for FFTs). | Supports standard linear, relative, and indexed addressing modes. |
| **Execution** | Deterministic, real-time instruction execution. | Optimized for average throughput, often non-deterministic due to cache misses or branch prediction. |

## 10. Applications of DSP Processors

> [!question] Question
> Mention three applications of DSP processors.

1. **Audio and Speech Processing:** Used for echo cancellation in VoIP, active noise cancellation (ANC) in headphones, and speech recognition engines.
2. **Digital Wireless Communications:** Handled in modems for modulation/demodulation, channel equalization, and OFDM in 5G and Wi-Fi networks.
3. **Biomedical Signal Processing:** Used in real-time filtering of ECG/EEG waveforms, hearing aids, and image reconstruction for MRI and CT scans.
